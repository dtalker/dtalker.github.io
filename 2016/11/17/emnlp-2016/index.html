Tutorials

T1：Practical Neural Networks for NLP: From Theory to Code

主要是介绍DyNet（CMU Group开发的一个NN库），在这个库上的一些常见网络结构以及NLP应用的一些实践实例。
其中库的

  • 核心：computation-graph formalism建模思路（所有的运算转化为DAG），
  • 优势：动态图定义（比如计算操作符的重载等）【相比于TensorFlow/Theano/Torch等】
  • project site: https://github.com/clab/dynet

基于DyNet介绍NLP中应用NN的常见网络结构，包括：MLP/CNN/RNN/LSTM.
My comments:

  • Part1较为基础，如果对NN比较熟悉可以跳过
  • Part2中的Tagging问题建议关注. 对话系统的NLU的Slot filling问题的一种经典建模方法（阿里用的就是Bi-LSTM）
  • 实用角度来说，静态定义的（tensorflow以及paddle）足够用

Sildes: Part1, Part2

T3: Lifelong Machine Learning for Natural Language Processing

Why Lifelong ML？

  • ML 1.0：传统机器学习主要是single-task learning，即每次学习都是重新利用数据
  • Human beings：人类在学习时会利用之前已有的知识，把之前的知识应用到新的任务上.
  • Key problem：leverage the prior knowledge learned from previous task（思想和transfer learning类似）

Algorithm Example：Lifelong Topic modeling

  • 核心：Topic Model中如何利用之前任务学习出的topic信息来提升当前任务下的效果
  • 问题关键1：之前任务学习到知识如何建模，1) must link{几个term必须在一个topic中} 2) Not link（几个term不在一个topic中）
  • 问题关键2：如何使用上述的知识？Gibbs sampling中通过改变抽样的比例来利用must link和not link信息

My comments：

  • LML这套在实际的工业应用场景中有，但是大多出现在冷启动的场景中（大部分冷启动场景主要也是迁移的是数据而非知识）
  • 将一个任务提取出的知识建模，而后融合到新的模型中的思路非常经典（如：现在NN模型上如何利用先验知识也是很火的方向）

Slides：https://pan.baidu.com/s/1jIEy3yQ

Invited Talks

Talk1: Learning in extended and approximate Rational Speech Acts models

By Christopher Potts (Stanford University)：http://web.stanford.edu/~cgpotts/
研究问题：在理性的对话沟通中的如何用合理的语言来有效表达信息,如：下图所示，如果想描述和表达中间那个头像，则选择glasses这个信息能比较清晰的传递信息

问题建模：RSA，信息接收者(Pragmatic listener)来说解读该信息取决三个方面：之前的状态（context），词汇信息（语言理解），对信息传递者的信息传递推理建模。

My comments：

  • 比较有意思的建模和研究，设计不少沟通的游戏有点类似经常玩的“我描述你猜“
  • 建模启发：人类在表达信息的时候会综合利用听众信息（听的人怎么理解）、语言信息、信息本身几个方面的要素，反过来接受者在decode时同样如此

Slides: https://pan.baidu.com/s/1kUZKT1t

Talk2：You Talking to Me? Speech-based and multimodal approaches for human versus computer addressee detection

by Andreas Stolcke(from MSR)
核心问题:基于语音/语言以及多模(图像)信号进行包含机器人的群聊场景中的”说话人”识别（区分是否该机器回答）
本文的研究，主要还是特征工程上的探索，主要的信号包括：

  • lexical info：N-gram class-specific LM
  • prosodic modeling（实验证明语音、语调等信号十分有用）：rhythmic、pitch/Using GMM
  • visual
  • spatial
  • Dialogue state

分类算法：LR

My comments: 实际的应用场景中这个问题多通过@（Google allo），或者唤醒语（比如 echo的alexa）来解决

Talk3：Learning Models of Language, Action and Perception for Human-Robot Collaboration

by Stefanie Tellex (Brown University)
核心问题：人机交互中机器如何对人的对话、动作进行识别、感知以及实现相应的任务
建模：POMDP(Partially Observable MDP)【也是经典的对话系统中对话管理建模】

  • State： Physical state(机器的物理状态)，human mental state（人的需求），Observation（通过这个建立state），physical sensor（对应physical state），Language（语言理解是人需求的一种表达），Gesture（手势也是人需求的一种表达）
  • Reward：是否完成任务
  • Actions：机器的物理动作(physical actions)、语言（Language）、手势(Gesture)


关键问题：mapping between human-language and object of the env（现实环境中的物体<->）
输入理解：word meanings from human’s speech（context是周围的环境）：P(y1,y2..| language)
输出生成(NLG)：对话时（比如澄清，哪个绿色的物件？）inverse semantics P(language | y1,y2,…)
Datasets collection：

  • Goal/command-based：给定一个视频（包含一些设定的动作，比如叉车取一个箱子到某一个位置上）让人描述一个指令

  • 收集渠道：amazon turk

    My comments：

    • 语言交互式机器人，impressive(有几个示例的视频： http://cs.brown.edu/~stefie10/ )
    • POMDP作为经典的建模方法，这里面的各个方面的因素考虑值得在对话系统中借鉴（如：上下文中的context类似这个问题中的环境context，而intent recognition和slot filling等问题则是在这个context下的semantical understanding）

Selected Papers

Papers – QA Session

Paper1： Creating Causal Embeddings for Question Answering with Minimal Supervision
问题：针对QA中的因果关系类型的问题进行针对性的优化
解决方案：

  • 利用pattern抽取高准确的因果关系context（句法分析 + 规则pattern）
  • 利用抽取到的因果关系context 训练specific word embedding（bi-directional embedding最有效）
  • 把上述word embedding的特征加入到QA的IR方法中，提升QA的相关性

Details： http://www.emnlp2016.net/program/392.html
My comments：细分子问题，构建子数据集来学习，是提升效果的一个比较经典的思路

Paper2： Improving Semantic Parsing via Answer Type Inference
问题：如何提升QA中问题中的语义解析准确率
解决方案：添加一个中间层信息（Answer Type类型）

  • 将Question句子转化为正常的表述语句，且抽取entity生成pattern
  • 基于上述表述语句，基于Bi-LSTM进行QA-Answer Type的分类

My comments：

  • Bi-LSTM用来做分类/Tagging已然是标配
  • 细分问题，增加一些中间信息可以提升效果，也是相对比较commo的思路
  • pattern的生成以及句式转化等细节是应用中的”trick”
    Details：http://www.emnlp2016.net/program/1072.html

Papers – Text Generation Session

Paper1：Generating Topical Poetry

问题：写诗问题（给定一个词在一个类型诗的字数、韵律等限制下，写一首语义有意境、相关联的诗）
挑战：诗体限制、语义关联、诗意等
解决方案（基于FSA进行严格的韵律等限制控制，利用RNN进行path选择）：

  • Choosing related words: using word embeddings
  • Selecting Rhyme pairs
  • Making a FSA（有限状态接收机）：生成一些候选
  • 基于RNN选择一条best path
  • 细节trick：对repeated words进行惩罚

数据集：歌曲（进行语义关联学习）
几个扩展问题

  • 如何把严格的格式和一些诗的限制放开，而让模型自己去学习结构
  • sentiment 信息加入
  • 行与行之间还只是在topic层面关联，不会有一些关联信息

details：http://www.emnlp2016.net/program/709.html

Paper2：Deep Reinforcement Learning for Dialogue Generation

问题：chi-chat中的对话生成
Previous work： seq2seq 生成式方法单轮的reward
Challenge

  • dull and generic responses
  • repetitive response（生成是问题容易陷入重复的回复中），Eg：See you later. -> See you later
  • short-sighted conversation decisions；Eg：How old are you? -> I’m 16 -> 16? -> ….

解决方案：RL model，将上述问题建模到reward中

  • State：previous two dialogue turns/使用LSTM模型学习的向量表示
  • Action： dialogue utterance to generate
  • Reward：Ease of answering：计算dull response在seq2seq model中当下产生的分数的负向；Information flow（diversity）：上下文语义相似的负得分；semantic coherence： Pseq2seq(s1|s2) + Pseq2seq(s2|s1)

RL训练：simulation
两个agent初始化时用common的seq2seq模型进行初始化
而后进行对话，基于RL模型进行模拟生成数据并学习
算法：AlphaGo-style strategy (Silver et al., 2016)
My Comments：

  • 闲聊中生成式的方法能保证召回和产生一些”意外”的回答，但是往往重复性等问题比较严重， 该方法基于RL的建模把一些问题通过reward建模，通过两个agent模拟来训练，是个不错的思路
  • 不过工业界还是IR路线为主（资源质量的可控，物料增加后的小范围泛化能力等）

details： http://www.emnlp2016.net/program/632.html

Paper3：Neural Text Generation from Structured Data with Application to the Biography Domain

问题：给定一些结构化的概念，生成一段整体的text(concepts->text)，文章聚焦在人物自传领域（已知一些结构化的个人信息，生成一段介绍的话）
算法核心：Neural Language Model for constrained sentence generation

  • 在NLM生成Text思想的基础上，将结构化的concept信息融合进去
  • 融合的方法：每个词除了有一个词典集上的index外，还有在concept table上的信息（filed【哪个字段】, position【该字段的那个position】）

  • 为了刻画一个filed结束的位置，position上既引入start的也引入end的position


    NN模型：经典的NLM网络结构（Embedding + 全连接层 + softmax层）
    Embedding： word embedding & table embedding(including the start position and end position<双向>)
    数据：wiki数据（描述文本 <-> 结构化concepts）
    details：http://www.emnlp2016.net/program/632.html
    My comments：table embedding的想法不错，这次会议里如何将结构化的信息融合到NN网络中的工作越来越多

Improving Information Extraction by Acquiring External Evidence with Reinforcement Learning
问题：信息抽取（IE）问题，如从一段刑事新闻中抽取凶杀者与凶杀的人数【KB构建的基础】
挑战：较强泛化能力的extractor需要大量的训练数据来cover住在不同context下的不同表达，如：shooter、suicide victims
解决方法：引入external的信息（如：基于该事件去search其他该事件的不同描述）
难点：

  • 如何生成引入external信息的查询query（直接title效果很差）【how to query】？
  • 如何融合新引入的外部信息抽取到的实体【how to reconcile】？
  • 引入external是否需要循环迭代？

算法： RL-based（将Query和Reconcile当做决策的action，将IE的accuracy作为reward来建模驱动）
State：包含几个信息组合的一个vector

  • 当前文章抽取的实体置信度 + 新文章抽取的实体置信度
  • 抽取到的实体的词其周边word的TF和IDF

  • 当前文章和新文章之间的文本相似性：IF/IDF based
    Action：两类决策动作（reconcile和query）

  • Reconcile：1） 接受新的某个实体的抽取（即信任某个新的实体抽取）2）接受所有新的 3）不接受所有新的 4）stop（即不再进行新的query和reconcile）

  • query： 候选集【一些模板和规则，主要围绕title和title的一些变种方法】
    Reward：刻画accuracy的变化以及同时对进行一次query查询进行惩罚
    Learning method: Deep Q-Network（Mnih et al.,2015 ）

Summary

整体的感受:

  • NN是标配算法
  • 如何将结构化的知识建模融合到DNN中是很多人在做的工作
  • RL method is blooming
  • 有一些问题在工业界并不是问题，或者说并不是那么急迫的问题，如：有不少paper在做群聊对话系统中，谁和谁对话，以及是否是机器对话（产品上可以用更为明确的方式达到相对OK的体验，比如 @,唤醒词等）
